## Sequencing Rationale

Three Key Technical Decisions from `spec.md` drive the order:

1. **Schema codegen first** (item 2) — every Swift file touching macros uses generated types from day one; zero schema drift.
2. **Engine before editor** (items 5–6 before 8) — the editor's "Run" button needs the engine; engine is testable against hand-authored YAML bundles without the editor.
3. **PS99 plugin authored last** (item 10) — engine + format must be game-agnostic; building PS99 first risks tight-coupling.

Item 6 (hand-authored test bundles) is a deliberate verification beat; it bridges items 5 (engine exists) and 7 (recorder exists) so the engine can be exercised against real fixtures before the recorder is built.
